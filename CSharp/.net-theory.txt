------------------------------ Features CIL , CLR , JIT , Garbage Collection

 1. Common Intermediate Language (CIL): When you compile C# code, it converts into an abstract, platform-independent bytecode called CIL. 
 
 2. Common Language Runtime (CLR): 
 The Microsoft .NET CLR is the virtual execution engine that runs this bytecode. 
 It handles essential tasks like security boundaries, type safety, and exception handling.

          The engine that runs .NET applications.
          
          Responsibilities:
          1, Memory Management (Garbage Collection)
          2, Exception Handling
          3, Security
          4. Thread Management
          
          Similar Node.js Runtime for JavaScript.
          
 3. Just-In-Time (JIT) Compilation: During execution, the CLR's JIT compiler translates the CIL bytecode into specific native machine instructions optimized for the host computer's CPU architecture.

 4. Garbage Collection (GC): Developers do not need to manually free memory. A background garbage collector automatically tracks object references and reclaims memory from objects that are no longer being used by the application, drastically reducing memory leaks.

4 ) Why was .NET Created?

Microsoft wanted a platform where developers could:
- Write code once and run on multiple systems
- Build enterprise applications faster
- Manage memory automatically
- Create secure and scalable applications
