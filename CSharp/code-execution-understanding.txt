 Step 3: CLR (Common Language Runtime)     
 CLR is the heart of .NET.

 Node.js Runtime = CLR

 when we run command 
    dotnet run

CLR starts.

1. CLR responsibilities:   
  Memory Management (create objects etc )
  Security - Checks permissions
  Exception Handling ( try catch )
  Thread Management
  Garbage Collection - Automatically frees memory.

2. Real Flow

   Program Starts
         ↓
   CLR Loads Assembly (.dll)
         ↓
   CLR Reads CIL
         ↓
   CLR Calls JIT
         ↓
   Native Code Generated
         ↓
   CPU Executes

------------------------

 Step 4: JIT (Just-In-Time Compiler)

 1. for example our code is 
   int a = 10;
   int b = 20;
   
   Console.WriteLine(a + b);
  
   CPU cannot understand CIL.
   CPU only understands: Binary instructions - eg. 10101010
  
   So CLR asks JIT to convert 
   CIL into Machine Code
-------------------
 2, 
 ex. C# > CIL > machine native code  
 C# code CSC Compiler converts C# to CIL 
 on dotnet build 

 and when we run notnet run 

     Assembly (.dll)
           ↓
     CLR Loads Assembly
           ↓
     CLR Reads CIL
           ↓
     CLR Invokes JIT Compiler
           ↓
     JIT Converts CIL → Native Machine Code
           ↓
     CPU Executes 
     -------------------------
           In Simple Way 
           Build Time
           ==========
           C# → CIL
           
           Run Time
           ========
           CIL → Machine Code

 JIT compiles only when needed.

 3. Why "Just In Time"?   -------- IMPORTANT
   Because compilation happens: 
   Just Before Execution , Not during build.

 4. Example:
      
      MethodA();
      MethodB();
      MethodC();
      
      If only MethodA runs:
      
      MethodA -> JIT Compile
      MethodB -> Not Compiled Yet
      MethodC -> Not Compiled Yet
      
      This saves memory.  

5. Example

    ----First call:
     
        Calculate();
     
    ---- CLR:
     
     Is machine code available?
           ↓
     No
           ↓
     Run JIT
           ↓
     Generate Machine Code
           ↓
     Store It
           ↓
     Execute
     
    ---- Second call:
     
     Calculate();
     
    ---- CLR:
     
     Machine code exists?
           ↓
     Yes
           ↓
     Execute Directly
     
  No recompilation.

Step 5: Garbage Collector (GC) : main reason to use .Net 

   In C/C++: 
   
        User* user = new User();
        
        delete user;
   
   Developer must manually free memory.
   If forgotten: Memory Leak
   
   but in In C#: 
       User user = new User();
       
    No delete needed.

  when we create 
      User user = new User();

  Memory:
     Heap
      └── User Object     

  Later : 
  user = null;   
  Now nobody references the object.
    Heap
     └── User Object (Unused)  

  GC notices and remove the unused 

\----Garbage Collector Flow

     Object Created
           ↓
     Application Uses Object
           ↓
     Reference Removed
           ↓
     GC Detects Unused Object
           ↓
     Memory Freed

=== Heap vs Stack

Very common interview question.

- Stack Stores:

int age = 25;
bool isActive = true;

Stack
 ├── age
 └── isActive

- Heap Stores:

User user = new User();
Heap
 └── User Object

GC works mainly on Heap.
